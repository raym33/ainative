Let's be honest

This project is in its earliest stages. We don't have a working OS yet. We don't have all the answers. What we do have is a clear vision and a commitment to building it in the open, with anyone who wants to help.

We're not promising to revolutionize computing next month. We're asking: what would an OS look like if AI was designed in from day one?

Vision

What we're trying to build

‚ö°

AI as Infrastructure

Not an app you install, but a system primitive. Like how Unix treats everything as a file, we want to treat AI capabilities as a fundamental resource available to every process.

üîí

Privacy by Default

Local inference first. Your data stays on your machine unless you explicitly choose otherwise. No phoning home, no cloud requirements for basic functionality.

üß©

Modular Architecture

Pick the components you need. Swap out the AI runtime, choose your UI paradigm, run on different hardware. Not a monolith, but a toolkit.

üîß

Developer First

If it annoys you, it's probably wrong. We're building for people who ship code, not for demo videos. Practical over impressive.

Contribute

We need all kinds of people

Systems Developers

C, Rust, Zig ‚Äî kernel hacking, driver development, low-level optimization. If you've written code that runs without an OS, we need you.

ML/AI Engineers

Model optimization, efficient inference, quantization, hardware acceleration. Make AI run fast with minimal resources.

UX Designers

Design interfaces that help, not annoy. What does an AI-native shell look like? How do we surface AI capabilities without being intrusive?

Technical Writers

Document everything. Architecture decisions, API references, tutorials. Good docs are the difference between a project and a community.

Researchers

What papers should we read? What prior art exists? Help us learn from what came before.

Skeptics

Poke holes in our ideas. Tell us what won't work. The best way to avoid building vaporware is to have people who'll call it out.
